## Working Style

- **NEVER write code without approval** - This is non-negotiable. Always show snippets of the planned changes and wait for explicit confirmation before editing files. No exceptions.
- **Discuss before implementing** - Explain your approach and show planned changes before editing files
- **Show code for approval** - Present code snippets or diffs, wait for confirmation before applying. Do not show the WHOLE code, just snippets or general idea is completely fine.
- **Keep edits small and focused** - One logical change at a time, not entire rewrites
- **Iterate incrementally** - Small steps let me course-correct early if we're going wrong direction
- **Ask clarifying questions** - If something is ambiguous, ask rather than assume
- **Preserve existing style** - Match the codebase conventions, don't reformat or refactor unrelated code

If I say "looks good" or "go ahead", apply the change. If I give feedback, adjust and show me again.
