Read the TODO file at: $@

Then work through the items one by one following these rules:

- Always start by reading the relevant source files to understand the current state. Do not read every file in the codebase to avoid polluting the context
- Present the issue and proposed fix before making any changes
- Wait for explicit confirmation before editing files
- Keep edits small and focused
- After completing an item, mark it with [X] in the TODO file
- After completing an item, suggest a commit message for the changes you made
- After completing an item, do not start the next one until confirmation hits
- If an item is deferred or skipped, mark it [X] and add a note explaining the decision
- Move to the next item only after the current one is resolved
- If unsure about an approach, ask clarifying questions rather than assuming
- Preserve existing code style and conventions
- At the end, show a summary table of all items and their resolutions
